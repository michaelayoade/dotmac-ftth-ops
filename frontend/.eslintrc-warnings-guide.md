## Warning Types

### 1. `react/no-unescaped-entities` - Unescaped Quotes/Apostrophes

**Issue**: React prefers HTML entities for quotes and apostrophes in JSX.

**Examples**:
```tsx
// ‚ö†Ô∏è Warning
<p>Don't use unescaped quotes</p>
<p>He said "hello"</p>

// ‚úÖ Fixed
<p>Don&apos;t use unescaped quotes</p>
<p>He said &quot;hello&quot;</p>

// ‚úÖ Alternative: Use curly braces
<p>{"Don't use unescaped quotes"}</p>
<p>{`He said "hello"`}</p>
```

**Quick Fixes**:
- `'` ‚Üí `&apos;` (apostrophe)
- `"` ‚Üí `&quot;` (double quote)
- Or wrap text in `{" ... "}` or ``{` ... `}``

### 2. `react-hooks/exhaustive-deps` - Missing Dependencies

**Issue**: React Hooks have dependencies that aren't listed in the dependency array.

**Example**:
```tsx
// ‚ö†Ô∏è Warning
useEffect(() => {
  loadData();
}, []); // loadData is missing from deps

// ‚úÖ Fix Option 1: Add dependency
useEffect(() => {
  loadData();
}, [loadData]);

// ‚úÖ Fix Option 2: Wrap in useCallback
const loadData = useCallback(() => {
  // ... load logic
}, []);

useEffect(() => {
  loadData();
}, [loadData]);

// ‚úÖ Fix Option 3: Disable if intentional
useEffect(() => {
  loadData();
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, []);
```

### 3. `@next/next/no-img-element` - Use Next.js Image Component

**Issue**: Using `<img>` instead of Next.js optimized `<Image>`.

**Example**:
```tsx
// ‚ö†Ô∏è Warning
<img src="/logo.png" alt="Logo" />

// ‚úÖ Fixed
import Image from 'next/image';
<Image src="/logo.png" alt="Logo" width={200} height={100} />

// ‚úÖ If external/dynamic image
{/* eslint-disable-next-line @next/next/no-img-element */}
<img src={dynamicUrl} alt="Dynamic" />
```

## CI/CD Configuration

To make builds fail on errors but pass on warnings:

```yaml

- name: Build
  run: pnpm build
  # Warnings won't fail the build (exit code 0)

- name: Strict lint (optional)
  run: pnpm lint --max-warnings 0
  # This would fail on any warnings
```
